## 작업 순서

1. `raw/` 폴더에서 가장 최근에 추가된 `YYYY-MM-DD.md` 파일을 찾아 읽어라.
2. 같은 스킬 폴더의 `template.md` 파일을 읽어 출력 형식을 파악하라.
3. raw 파일의 내용을 `template.md` 형식에 정확히 맞춰 한국어로 요약하라.
4. 결과를 `summary/YYYY-MM-DD.md` 경로로 새 파일을 생성하고 커밋·푸시하라.
   - 날짜는 raw 파일과 동일하게 사용
   - 커밋 메시지: `brief: YYYY-MM-DD`

## 요약 원칙

- 원문에 없는 사실을 추가하지 말 것
- 출처 링크는 raw 파일에 있는 URL을 그대로 사용
- 템플릿에 정의된 섹션 외 다른 내용을 추가하지 말 것
- "주요 이슈 TOP 3"는 비즈니스 의사결정 관점에서 중요도 높은 순으로 선정
- "시사점"은 raw에 명시된 사실에 근거해서만 작성, 추측·과장 금지

## 금지 사항

- 템플릿 형식 임의 변경 금지
- summary 파일에 메타 정보(작성자, 모델명 등) 추가 금지
- raw 파일 수정·삭제 금지
